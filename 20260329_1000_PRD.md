## 1. Resumen ejecutivo

Aplicación móvil Flutter para gestionar tarjetas de asistencia a clases de baile del estudio **Evolución • Baila Más**. Reemplaza las tarjetas físicas de cartón por una versión digital interactiva que permite registrar clases con fecha y horario, compartir la tarjeta por WhatsApp y persistir los datos entre sesiones.

---

## 2. Problema

Los alumnos del estudio usan tarjetas físicas de cartón para registrar su asistencia a clases. Este sistema tiene varias limitaciones:

- Las tarjetas se pierden o deterioran fácilmente.
- No hay forma de compartir el estado de la tarjeta de forma remota.
- No queda registro del horario exacto de cada clase tomada.
- Un alumno puede tomar más de una clase por día (distintos horarios) y la tarjeta física no permite distinguirlas.
- El administrador del estudio no puede consultar el estado de las tarjetas sin tener el cartón en mano.

---

## 3. Objetivo del producto

Proveer una app móvil simple y visual que:

1. Reproduzca fielmente la experiencia de la tarjeta física.
2. Permita registrar cada clase con fecha y horario seleccionado.
3. Persista los datos localmente para que no se pierdan al cerrar la app.
4. Permita compartir la tarjeta como imagen por WhatsApp u otras apps.

## 5. Alcance — versión 1.0

### 5.1 Funcionalidades incluidas

#### Tarjeta digital
- Visualización de tarjeta con gradiente rosa/naranja/violeta que imita la tarjeta física original.
- Muestra nombre del alumno, nombre del estudio y pack de clases contratado.

#### Gestión del nombre
- El alumno puede editar su nombre y apellido directamente en la tarjeta.
- El nombre se guarda automáticamente en el dispositivo.

#### Gestión del pack de clases
- El alumno puede seleccionar entre 3 opciones de pack: **4, 8 o 16 clases**.
- Al cambiar el pack, si hay clases marcadas, la app solicita confirmación antes de resetear los slots.
- El pack seleccionado se guarda automáticamente.

#### Registro de clases
- Cada clase se representa como un slot (círculo) en una grilla de 2 columnas.
- Al tocar un slot vacío, se abre un selector de horario con 3 opciones fijas:
  - **17:00**
  - **19:30**
  - **20:30**
- Al confirmar el horario, el slot se marca mostrando la **fecha** (formato `d/M`) y el **horario** elegido.
- Soporta múltiples clases por día en distintos horarios, ya que cada slot guarda su propio timestamp.
- Al tocar un slot ya marcado, la app solicita confirmación para desmarcarlo.

#### Compartir tarjeta
- Botón flotante "Enviar por WhatsApp" que captura la tarjeta como imagen PNG.
- Se abre el menú nativo de compartir del sistema operativo.
- El usuario elige la app de destino (WhatsApp, correo, etc.).
- La imagen se genera con alta resolución (pixelRatio 2.0).

#### Persistencia de datos
- Todos los datos (nombre, pack, slots marcados con fecha y horario) se guardan automáticamente en `SharedPreferences`.
- Al abrir la app, se restaura el último estado guardado.
- Si no hay datos guardados, se inicializa con un pack de 8 clases vacío.

### 5.2 Funcionalidades excluidas en v1.0

- Múltiples tarjetas por alumno.
- Sincronización con backend o base de datos remota.
- Notificaciones push.
- Historial de tarjetas anteriores.
- Modo administrador / panel de control.
- Autenticación / login.

## 7. Arquitectura

```
lib/
├── main.dart                          # Entry point, ProviderScope
├── models/
│   └── class_card_model.dart          # ClassSlot, ClassCardModel (toJson/fromJson)
├── services/
│   └── storage_service.dart           # Lectura/escritura en SharedPreferences
├── providers/
│   ├── storage_provider.dart          # Provider de SharedPreferences y StorageService
│   ├── storage_provider.g.dart        # Generado por build_runner
│   ├── class_card_provider.dart       # ClassCardNotifier (AsyncNotifier)
│   └── class_card_provider.g.dart     # Generado por build_runner
└── screens/
    └── class_card_screen.dart         # UI principal (ConsumerStatefulWidget)
```

### Patrón de datos

```
UI (ref.watch) ──→ ClassCardNotifier ──→ StorageService ──→ SharedPreferences
                         ↑
               ref.read(...notifier).método()
```

- El modelo es **inmutable** (usa `copyWith`).
- El notifier es **AsyncNotifier** porque `build()` lee de disco de forma asíncrona.
- Cada modificación actualiza el estado en memoria y persiste en disco en la misma operación.

## 8. Modelo de datos

### ClassSlot
```
date  : DateTime?   — fecha en que se tomó la clase (null = vacío)
time  : String?     — horario elegido ("17:00" | "19:30" | "20:30")
isUsed: bool        — computed: date != null && time != null
```

### ClassCardModel
```
studentName  : String         — nombre y apellido del alumno
totalClasses : int            — cantidad de clases del pack (4 | 8 | 16)
slots        : List<ClassSlot> — uno por cada clase del pack
usedCount    : int            — computed: cantidad de slots usados
hasMarkedSlots: bool          — computed: si hay al menos un slot marcado
```

## 9. Flujos principales

### 9.1 Marcar una clase
1. Usuario toca un slot vacío.
2. Se abre bottom sheet con 3 botones de horario.
3. Usuario elige el horario.
4. El slot se marca con la fecha actual y el horario elegido.
5. El estado se guarda automáticamente en disco.

### 9.2 Desmarcar una clase
1. Usuario toca un slot marcado.
2. Se muestra diálogo de confirmación con la fecha y horario de la clase.
3. Usuario confirma.
4. El slot vuelve a estado vacío y se guarda en disco.

### 9.3 Cambiar pack
1. Usuario toca "✏ editar" junto al label del pack.
2. Se abre bottom sheet con las 3 opciones (4, 8, 16).
3. Si hay clases marcadas, se muestra advertencia en el sheet.
4. Si el nuevo pack es distinto al actual y hay clases marcadas → diálogo de confirmación.
5. Al confirmar: se resetean todos los slots y se guarda en disco.

### 9.4 Compartir tarjeta
1. Usuario toca el FAB "Enviar por WhatsApp".
2. La app captura el widget de la tarjeta como imagen PNG (pixelRatio 2.0).
3. La imagen se guarda temporalmente en el directorio temporal del dispositivo.
4. Se abre el menú nativo de compartir con el texto "🎵 Tarjeta de clases de [nombre]".
5. El usuario selecciona WhatsApp (o cualquier otra app).

## 10. UI / UX

### Paleta de colores
| Elemento | Color |
|---------|-------|
| Gradiente inicio | `#F5C842` (amarillo) |
| Gradiente medio | `#E0608A` (rosa) |
| Gradiente fin | `#9B4DB0` (violeta) |
| Texto slot marcado | `#9B4DB0` |
| Horario slot marcado | `#C06AB0` |
| FAB | `#9B4DB0` |

### Componentes clave
- **Tarjeta**: `Container` con `LinearGradient` y `BorderRadius` de 20px.
- **Slots**: `AnimatedContainer` con `BorderRadius` circular. Blanco sólido si marcado, blanco semitransparente si vacío.
- **Selectores**: `ModalBottomSheet` con `RoundedRectangleBorder` (top radius 20px).
- **Confirmaciones**: `AlertDialog` estándar de Material.
- **FAB**: `FloatingActionButton.extended` con ícono de compartir.
